# Environment Variables

This page separates variables consumed by the current local product from variables reserved for provider-backed integrations.

## Credential Requirement Matrix

| Scenario                           | Minimum variables                                                                                                                                                     | Notes                                                                                                                                                                                                                  |
| ---------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Local seeded demo                  | None                                                                                                                                                                  | Uses defaults and in-memory data.                                                                                                                                                                                      |
| Admin-authenticated API            | `BEK_ADMIN_API_TOKEN`, `BEK_REQUIRE_ADMIN_AUTH=true`, optional runtime entry in the web console                                                                       | Required outside explicit local demo mode. `VITE_BEK_ADMIN_API_TOKEN` is dev-server-only; production web builds reject embedded admin tokens.                                                                          |
| Signed Slack callbacks             | `SLACK_SIGNING_SECRET`                                                                                                                                                | Required for events, commands, and interactivity.                                                                                                                                                                      |
| Slack OAuth install redirect       | `SLACK_CLIENT_ID`, `SLACK_STATE_SECRET`, `SLACK_REDIRECT_URI`                                                                                                         | Builds signed Slack authorize URLs for the raw endpoint and the web Slack install action.                                                                                                                              |
| Slack OAuth callback/exchange      | `SLACK_CLIENT_ID`, `SLACK_CLIENT_SECRET`, `SLACK_STATE_SECRET`, `SLACK_REDIRECT_URI`, `BEK_CREDENTIAL_MASTER_KEY`, `BEK_SLACK_OAUTH_EXCHANGE=true` outside production | Code exchange stores bot tokens in the local encrypted vault. If exchange is disabled, callbacks validate state and return without storing a token. Hosted installs still need managed KMS/broker custody.             |
| Slack Web API posting              | Stored OAuth token or `SLACK_BOT_TOKEN`                                                                                                                               | Self-hosted/local posting. `SLACK_BOT_TOKEN` is a manual fallback when OAuth token persistence is not used.                                                                                                            |
| Durable self-hosted worker         | `BEK_STORAGE=postgres`, `DATABASE_URL`, `BEK_RUN_ADVANCEMENT=worker_local`, `BEK_WORKER_QUEUE_BACKEND=postgres`                                                       | Persists worker records, leases, dead letters, and worker events. Run `pnpm db:migrate` before boot.                                                                                                                   |
| GitHub App validation and webhooks | `GITHUB_APP_ID`, `GITHUB_APP_PRIVATE_KEY`, `GITHUB_APP_WEBHOOK_SECRET`, optional `GITHUB_APP_INSTALLATION_ID`                                                         | Validation helpers, setup preview, and signed `/api/github/webhooks` ingress exist; real GitHub API writes/token exchange are not wired. `GITHUB_WEBHOOK_SECRET` is a deprecated alias.                                |
| Model provider calls               | `BEK_MODEL_GATEWAY=vercel_ai_sdk`, `AI_GATEWAY_API_KEY` or `VERCEL_OIDC_TOKEN`, optional `BEK_AI_GATEWAY_TAGS`                                                        | Active for the AI SDK Gateway runtime. Default `BEK_MODEL_GATEWAY=local` keeps zero-config demos deterministic.                                                                                                        |
| MCP tools                          | `MCP_CONFIG_PATH`, `MCP_GATEWAY_ENCRYPTION_KEY`                                                                                                                       | Reserved until live transport and credential handling are implemented.                                                                                                                                                 |
| Local Docker sandbox provider      | Docker installed on the host, `BEK_RUN_ADVANCEMENT=worker_local`, `BEK_SANDBOX_PROVIDER=docker-local` or `docker`                                                     | Local worker runs the `opencode-sandbox` profile through `DockerSandboxProvider`. This is a sandbox-command adapter, not full OpenCode repo orchestration yet. Compose leaves it off with `BEK_SANDBOX_PROVIDER=none`. |
| Hosted sandbox execution           | `BEK_SANDBOX_PROVIDER`, plus provider-specific credentials such as `E2B_API_KEY` or `VERCEL_OIDC_TOKEN`                                                               | Reserved until hosted sandbox providers are wired end to end.                                                                                                                                                          |

## Model Execution Env Notes

Provider/model strings such as `openai/gpt-5.4` and
`anthropic/claude-sonnet-4.8` are stored in Bek model policies. They match the
Vercel AI Gateway and AI SDK string shape. The deterministic local runtime is
still the default; live Gateway execution requires
`BEK_MODEL_GATEWAY=vercel_ai_sdk`.

For Gateway execution:

- Prefer `AI_GATEWAY_API_KEY` for local, CI, or non-Vercel deployments.
- Prefer `VERCEL_OIDC_TOKEN` for Vercel deployments or local Vercel-linked
  development where OIDC is available. Local OIDC tokens must be refreshed when
  they expire.
- Do not put provider-specific keys, Gateway keys, or OIDC tokens in prompts,
  run artifacts, sandbox env, screenshots, fixtures, or issue reports.
- Treat `model.completed` events as the source for provider/model, response
  usage, latency, Gateway response ID, estimated cost, and local actual
  estimate. The local actual is calculated from response usage and Bek's pricing
  benchmark; it is not the provider-billed amount.
- Reconcile the durable `model_usage` ledger against Gateway/provider dashboards
  as a separate process. In Postgres mode, `/api/model-usage` totals come from
  the durable ledger and return `source: "model_usage"`; in memory mode they
  fall back to run-level fields with `source: "runs"`. Neither source is
  provider-billed evidence until reconciliation is added.

## Secret Handling

- Do not commit `.env` files, Slack secrets, model provider keys, GitHub tokens, or sandbox credentials.
- Do not paste raw secrets into prompts, fixtures, screenshots, logs, issues, or docs examples.
- Prefer a platform secrets manager for hosted or shared deployments.
- Treat Slack payloads, MCP tool descriptions, repository files, and model output as untrusted input.
