# Code Of Conduct

Bek is open-source infrastructure for agent teammates. The project should be a place where contributors can discuss product, security, and engineering tradeoffs without personal attacks or gatekeeping.

## Expected Behavior

- Be respectful and direct.
- Assume good intent, but accept impact when someone points it out.
- Keep criticism focused on code, docs, design, threat models, and product behavior.
- Help new contributors understand Bek's core invariant: one visible `@bek` teammate with admin-governed capabilities behind it.
- Avoid posting secrets, private workspace data, private prompts, or private repo contents.

## Unacceptable Behavior

- Harassment, threats, insults, or discriminatory language.
- Sexualized attention or imagery.
- Doxing or publishing private contact information.
- Repeated disruption after maintainers ask for a change in behavior.
- Publicly posting security vulnerabilities before maintainers have had a chance to respond.

## Reporting

Report conduct issues privately to `conduct@bek.dev`. Include links,
screenshots, or other context when it is safe to share them.

Security vulnerabilities should follow `SECURITY.md` and be reported privately, not through a public issue.

## Enforcement

Maintainers may remove comments, close issues, block users, or revoke project access when needed to keep the project healthy. Enforcement should be proportionate, documented privately when appropriate, and focused on protecting contributors and users.
